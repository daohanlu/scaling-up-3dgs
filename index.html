We present Grendel, a distributed training system, to partition 3D Gaussian Splatting (3D GS) parameters and parallelize its computation across multiple GPUs. We explore different learning rate scaling strategies and identify the simple √(batch_size) scaling rule to be highly effective. Using Grendel, we show that scaling up 3D GS training in terms of parameters and compute leads to significant improvements in visual quality on multiple large-scale scene reconstruction datasets.

Introduction

3D Gaussian Splatting (3DGS)\cite{3dgs} has been an emerging and popular technique for novel 3D view synthesis. Its popularity is because it offers faster training and rendering than comparable previous approaches such as NeRF~\cite{nerf}. However, most existing 3DGS pipelines are limited to using a single GPU for training. As single memory and computation constraints become a bottleneck when applying 3DGS to higher-resolution or larger-scale scenes. For example, the standard Rubble dataset~\cite{meganerf} contains 1657 images, each with a 4K resolution. On a single A100 40GB GPU, the maximum number of Gaussians that can be processed is limited to 11.2 million. However, this number is well below the saturation point for 3DGS, and, as we show in \ref{sec:n3dgs-reconstruction-quality}, increasing the number of Gaussians improves rendering quality (as measured using the rendered image's PSNR). Thus, to make 3DGS more widely applicable we need to develop approaches that scale 3DGS to multiple GPUs, and understand 3DGS' scaling behavior.
